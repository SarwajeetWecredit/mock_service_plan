🎯 Complete Control Flow

BA Controls: Admin Override → System Processes: UAT Form → Result: Custom Response Delivered

📋 Step-by-Step Process

1️⃣ Setup Admin Override

BA Actions:

  1. Open Admin UI dashboard
  2. Choose lender (ABFL/TrustPaisa/Fibe)
  3. Enter any mobile number for testing
  4. Configure expected response
  5. Set usage limits and delays
  6. Save override
2️⃣ Test on UAT Website

BA Actions:

  1. Go to UAT website: https://uat.wecredit.co.in
  2. Fill loan application form
  3. Use the same mobile number from admin override
  4. Complete other form fields normally
  5. Click "Apply Now" or "Submit"
  6. Observe the response
3️⃣ Validate & Document

BA Actions:

  1. Check if response matches admin override
  2. Verify message content and formatting
  3. Test response timing (delays)
  4. Take screenshots for documentation
  5. Record Pass/Fail status
  6. Clean up admin override if needed

🎛️ What BA Can Control via Admin UI

🎯 Target Selection

UI Controls:

  • Lender Dropdown: ABFL, TrustPaisa, Fibe
  • Mobile Number Input: Any 10-digit number
  • Endpoint Selection: Specific API endpoint (optional)

What it does: Defines which mobile number gets custom response from which lender

💬 Response Control

UI Controls:

  • Status Code Dropdown: 200 (Success), 400 (Bad Request), 500 (Server Error)
  • Response Type: Success, Rejection, Error, Custom
  • Message Text Area: Custom message user will see
  • Response Template: Pre-built templates for common scenarios

What it does: Controls exactly what message and status the user sees on UAT website

⏱️ Timing & Usage

UI Controls:

  • Response Delay Slider: 0ms to 30 seconds
  • Usage Limit Dropdown: 1, 5, 10, Unlimited times
  • Enable/Disable Toggle: Activate or deactivate override
  • Expiry Date Picker: Auto-cleanup after date

What it does: Controls response timing, how many times it works, and when it expires

📊 Monitoring & Management

UI Controls:

  • Active Overrides List: View all current overrides
  • Usage Counter: See how many times used
  • Edit Button: Modify existing overrides
  • Delete Button: Remove overrides
  • Bulk Actions: Clean up multiple overrides

What it does: Lets BA manage and track all test scenarios
